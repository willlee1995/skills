# SOW Clauses (full)

## Revision section

```
REVISIONS & APPROVALS

1. Included rounds. This engagement includes [2] rounds of revisions
   on the agreed deliverables.

2. Definition of a round. A "round" is a single, consolidated set of
   feedback gathered from all stakeholders, delivered in one document,
   and addressed by us in full before the next version is issued.
   Feedback delivered piecemeal or across separate messages may be
   treated, at our discretion, as separate rounds.

3. What is a revision (included). Adjustments to elements already
   approved at brief and storyboard stage: timing and pacing, colours
   within the agreed palette, approved copy edits, and corrections of
   errors or technical glitches.

4. What is a new request (not included). New scenes; additional
   runtime; new creative concepts or directions; additional formats,
   durations, or aspect ratios; assets not previously supplied; and
   any feedback reversing a previously approved decision. New requests
   are handled as change orders (clause 7).

5. Additional rounds. Revision rounds beyond those included are billed
   at [day rate / fixed fee per round] and may extend the timeline.

6. Approval gates. Work proceeds through sign-off stages: brief →
   storyboard/animatic → first animation → final. Approval at each gate
   locks that stage. Reopening a locked stage is a new request.

7. Change orders. Any new request will be quoted (cost + timeline
   impact) in writing before work proceeds. Work on the new request
   begins only on written approval.
```

## Approval-gate language (standalone)

```
SIGN-OFF GATES
Each stage below must be approved in writing before the next begins.
Approval locks that stage:
  • Brief — creative intent and message.
  • Storyboard / animatic — structure, scenes, pacing, copy.
  • First animation — motion, timing, colour.
  • Final — delivery-ready files.
Changes to a locked stage are change orders, not revisions, because
later work was built on the approved version.
```

## "Round" definition variants

- **Strict**: "A round is one consolidated feedback document. Separate messages count as separate rounds." (Use with clients prone to drip-feeding.)
- **Standard**: "A round is one consolidated set of feedback addressed in full." (Default.)
- **Generous**: "Minor follow-ups within 24h of returning notes fold into the same round." (Use for trusted, long-term clients.)

## Kill / pause clause (optional)

```
If the project is paused by the client for more than [10] working days,
remaining work is rescheduled subject to availability, and the balance
for work completed to date becomes payable.
```
