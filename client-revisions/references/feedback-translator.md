# Feedback Translator (expanded)

Never touch the project file on a vague note. Convert it to a decidable choice and ask for a timestamp.

## Lookup table

| Client says | Likely axes | Ask back |
|---|---|---|
| "Make it pop" | colour / pace / scale / sound | "Saturation, faster cuts, bigger scale, or louder sound? Point me to a second that already pops." |
| "It feels off" | pace / colour / type / message | "Off in pacing, colour, type, or message — and which 2 seconds feel worst?" |
| "More premium" | easing / space / palette / type | "Slower easing, more breathing room, restrained palette, or refined type?" |
| "More dynamic" | camera / cuts / motion density | "More camera movement, faster cuts, or more elements moving?" |
| "I don't love it" | unknown — needs anchor | "What specifically? And a reference for what 'loving it' looks like?" |
| "Punch it up" | pace / colour / sound | "Pacing, colour, or sound? Show me a moment with the energy you want." |
| "Make it bigger" | scale / weight / contrast / time | "Bigger scale, bolder type, more contrast, or more screen time?" |
| "Something's missing" | scene / message / brand / energy | "A scene, a message, a brand element, or energy — and where?" |
| "Too busy" | element count / pace / colour | "Fewer elements at once, slower pace, or simpler palette?" |
| "Cheap-looking" | type / spacing / easing / colour | "Usually type choice, spacing, easing, or colour — which feels cheapest to you?" |
| "Not on brand" | colour / type / logo / tone | "Which — colour, type, logo usage, or overall tone?" |
| "Make it flow better" | transitions / pacing / continuity | "Smoother transitions, steadier pace, or stronger scene-to-scene continuity?" |

## The "problem, not fix" coaching note for clients

Share this once, early, to raise feedback quality:

> The most useful feedback names the problem, not the solution. "The
> intro drags" lets me fix it the right way; "make the intro faster"
> locks me into one fix that might not be the real issue. Tell me what
> feels wrong and where (a timestamp), and leave the how to me — you'll
> get a better result in fewer rounds.

## Three questions that decode almost any vague note

1. **Where?** — "Which timestamp or scene?"
2. **Which axis?** — colour, pace, type, scale, sound, or message?
3. **Compared to what?** — "Point to a reference, or a second in this video that already feels right."
