## Instructions for the agent

1. Use **Part A–C** for requester software justification; leave **Part D–F** for department head and ITD unless the user supplies approver details.
2. **Any Data access by this Software — Yes**: Page 2 (*Privacy Data Measurement*) Sections 1–3 **must** be answered in substance—not boilerplate only. If the filled PDF shows **Yes** but page 2 questions are empty, **flag this** and prompt the user to complete privacy sections before submission.
3. **Any Data access — No**: Page 2 usually left blank or marked N/A per local practice—confirm with requester.
4. **Software Type** for Acrobat Pro: prefer **Office Automation Tool** or **Graphic Tool** per unit convention; **Others** only if IT specifies.
5. Attach **licence / PO / subscription** evidence as Part B requests (“Please attach license information”). Call out **user-purchased** licences explicitly in Remarks when applicable.
6. Never invent **PC Serial No., PC Asset ID, Computer Name, PO No.**—use placeholders only if the user explicitly asks for placeholders.

## Typical Acrobat Pro wording (Radiology / consent PDFs)

- Remarks often cite **user-purchased** licence and purpose such as **editing PDF consent forms** or general PDF editing—mirror the user’s exact clinical/non-clinical wording.
- Subscription/request dates sometimes coincide (**same calendar date**) when renewing annually—verify with the user.

## Page 2 (Privacy Data Measurement)

If **Yes** to data access:

- Section 1: Types of data input (e.g. identifiers vs generic documents), access controls, storage location (PC/mobile/cloud).
- Section 2: Retention, backup, encryption, mobile involvement.
- Section 3: Standalone workstation policy, end-user guidance.

Keep answers proportionate to actual risk; avoid claiming “no patient data” if consent forms hold identifiable information—align wording with hospital privacy rules.

## Checklist before handoff

- Part A: name parts, department, rank, office, phone, date.
- Part B: software name, vendor, remarks + licence route; subscription/licence evidence referenced or attached.
- Part B checkbox row and **Any Data access** choice consistent with page 2 completion.
- Part C: copyright line acknowledged; signature/date when submitting physically or digitally.
- Page 2: completed **if and only if** “Any Data access” is Yes (otherwise explicitly N/A per local rule).

## Related notes

- Freeware / non-licensed software: consult **local IT** before submission (printed note on form).
- Intellectual Property Ordinance note appears on page 2—lawful licence required.
